# Terrane 质量评测集设计（PRD §2/§11 验收依据）

## 1. 引用准确率评测集（每版本发布前人工判定）
- 规模：≥200 条（中文 120 / 英文 80）；语料 = 出厂示例库 + 3 个真实领域包（法律案例/学术论文/产品文档，各 ~30 源，脱敏）
- 每条 = {问题, 期望答案要点, 必须命中的源 chunk}；判定：答案每个事实陈述的引用是否指向支持该陈述的 chunk（二人判定，分歧仲裁）
- 指标：引用准确率 ≥95%；无引用陈述必须带"推断"标注（漏标率 <2%）
- 同集复用于检索回归：top-20 召回命中率按档位（小库/向量）分别报

## 2. Lint 评测集（种子库法）
- 构造：在干净示例库人工埋入——矛盾陈述对 ×15、过时主张（新源推翻旧源）×10、孤立页 ×10、缺专属页概念 ×10、缺失交叉引用 ×15
- 指标：各类检出率（目标 ≥80%）+ 误报率（≤15%）；每版本跑全量并公布
- 修复质量抽检：一键修复的 diff 人工抽 20 项判定正确性

## 3. 记忆评测集（自建，不采信 LoCoMo）
- 构造：50 段多轮对话脚本（含事实陈述/偏好/纠正/矛盾更新），金标 = 期望抽取的记忆条目与 ADD/UPDATE/DELETE/NOOP 决策
- 指标：抽取召回 ≥80%、消解决策准确率 ≥85%、时序层 as-of 查询正确率 100%（PoC 已验语义，此处验管线）
- 唤回评测：20 个"新源应唤回旧记忆"场景，命中率 ≥70%（阈值默认 0.75 下）

## 4. R4 抽取质量集（离线包 CPU 档门槛）
- 见 poc/harness/r4_extract_quality.py：30 条中英段落金标；实体召回 ≥0.8 / 关系召回 ≥0.7 / JSON 合法率 ≥0.98 → 时序层 CPU 档可开，否则降级文案
- 维护：评测语料入 `terrane-shared/evals/`（仓库内版本化）；CI 在阶段⑤后接入自动跑 1/3 子集做回归闸门
