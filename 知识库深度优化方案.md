## 一、解析引擎做到 SOTA（你的点 1）

**现状**：解析引擎是**纯词法**（PyMuPDF 几何算法抽文本/表格/版面）+ 仅对「扫描页/嵌入图片」过 VL。表格靠坐标聚类、公式靠字体启发式、图表只描述不提数据、视频是固定间隔抽帧。

**业界 SOTA（调研）**：Docling（DocLayNet 版面模型 + TableFormer 表格，复杂表 97.9%）、MinerU（LayoutLMv3 版面 + YOLOv8 检测 + UniMERNet 公式 + 84 语种 OCR）、Marker、Qwen-VL「QwenVL-HTML」版面解析模式。核心范式 = **版面理解 → 分块 → 按块类型专门处理（表格/公式/图/正文）→ 阅读顺序重组**。

**方案（分层，全部优雅降级，无模型仍可用）**：
1. **版面感知主路**：整页走 VL「版面解析」(QwenVL Markdown/HTML 模式) → 高保真 Markdown（标题层级、阅读顺序、分栏正确）。复杂版面页启用，纯文本页走快速词法（平衡成本/速度）。
2. **表格**：复杂表（跨行列合并/无框线）→ VL 转结构化 Markdown 表格；简单表保留词法。
3. **公式**：图片型/复杂公式 → VL 转 LaTeX（`$$...$$`）。
4. **图片/图表**：描述 + 图表数据提取（图表→数据表）。
5. **扫描件/图片 PDF**：整页 VL OCR（已做，强化 prompt + 分块）。
6. **视频升级**：场景切分（scene detection）+ 关键帧 VL 描述 + ASR 词级时间戳转录 + 章节化（对标 SceneRAG/VideoRAG），取代现固定抽帧。
7. **音频**：ASR + 章节/说话人分段。
8. **三档解析档位**：快速（纯词法）/ 标准（词法+VL图片，现状）/ 高精（整页 VL 版面）。上传时可选或后台设默认。
9. **结构化输出**：每块保留页码/bbox/类型元数据 → 支撑「点引用跳原文高亮」（见第三项）。
10. **质量项**：页眉页脚去除、目录/参考文献识别、分栏边界、阅读顺序。

**我的推荐**：以「VL 版面解析为主路 + 词法兜底 + 三档可选」重构引擎，视频升级为场景化。这是把引擎做到 SOTA 的核心。**注意：整页 VL 会增加 token 成本**（见决策②）。

## 二、真·动态局部加载（你的点 2）

**根因（已定位）**：懒加载组件 `LazyPageViewer` **本身是真按页 IntersectionObserver 懒加载的**——问题不在前端。真正卡点在后端 `render_service.render_source_bg`：**把所有页渲染完才把 status 从 rendering 改成 done**，前端在 done 之前一直显示「加载中/生成版面预览」。512 页要等约 2 分钟全渲完才一次性出现 → 你看到的「全量加载」。

**方案**：
1. **渐进渲染**（立即改善）：每渲好一页（或每 N 页）就增量写 `raw_source_renders.pages` + `page_count`，rendering 期间前端即可拉已完成页并显示；进度条「已生成 X/Y 页」。
2. **虚拟滚动**：`LazyPageViewer` 用虚拟列表（@tanstack/virtual），不一次性挂几百个 DOM 占位节点。
3. **按需渲染**（彻底）：大文档（>50 页）首屏只渲前几页，滚到第 N 页才 `POST request-render/{n}` 渲该页 → 真正秒开。

**我的推荐**：渐进渲染 + 虚拟滚动（必做）；大文档叠加按需渲染（彻底解决）。

## 三、预览页内「文档级 AI 问答」（你的点 3）

**方案**：预览页右侧（或独立面板）加 `DocumentSourceChat` —— 针对**当前这一个文档**问答，答案带引用，引用可点跳到原文对应页高亮（依赖第一项的 bbox 元数据）。后端 `search_chunks` 加 `raw_source_id` 过滤、chat/search 端点加 `source_id` 参数（已定位改造点）。

## 四、修复「直接问找不到文件」+「点文件→基于该文档问答」（你的点 5）

**根因（已定位）**：检索函数 `search_chunks()` **没有按文档过滤的参数**，且全库检索；问得泛时词法+向量两路都不中 → 报「找不到文件」，得在提示词里写文件名才间接收窄。

**方案**：
- 加 `raw_source_id` 过滤 → 点某文件即「基于该文档问答」，无需写文件名。
- KB 主页直接问也修：提高召回（降低相似度阈值/扩召回数）；没命中时给**明确空态引导**（"未在本库找到相关内容，试试换个说法或指定文档"）而非生硬「找不到文件」。

## 五、预览页「对话 / 解析对比」模式切换（你的点 6）

**方案**：预览页顶部加**分段控件** `[ 对话 ] [ 解析对比 ]`。
- **对话**：全宽「基于本文档问答」+ 引用跳页。
- **解析对比**：现有「左原文版面 / 右解析结果」。
- 默认模式可配（见决策③）。

## 六、知识库页面 SaaS 级视觉/体验重做（你的点 4）

**现状短板（调查）**：列表卡片过简无缩略图/元信息；详情页源列表过窄（20rem）、无搜索排序；tab（chat/search/studio/wiki/graph）是圆形按钮不突出；Chat 引用 badge 过小且**不可点跳原文**；搜索结果无排序；移动端分栏破碎；空态引导弱。

**方案**：
- **列表**：卡片加缩略图 + 元信息（源数/大小/最近更新）+ 最近查看。
- **详情页**：源列表加宽 + 搜索/排序/状态过滤 + 处理进度实时动画；tab 改 segment control + 图标。
- **Chat**：引用来源可点 → 侧滑/跳原文高亮；流式「typing」动画；清空历史。
- **搜索**：结果按相关度/日期排序 + 高亮命中。
- **响应式**：移动端纵向分屏 + 触控翻页；图谱触控支持 + 图例 + 节点搜索。
- **空态**：每个空态给下一步引导。
- 统一设计系统（按钮/卡片/Tabs/Modal 复用组件）。

## 七、移除 License 激活（你的点 7，开源）

**全触点已定位**（后端中间件 `license_gate.py`/`licensing/state.py`/`api/license.py`/`vendor/forge_verifier`、admin 激活 API + 向导、前端 `RequireLicense`/`LockedPage`/`ActivatePage`/激活徽章、gateway Go `internal/license`、部署 `TERRANE_LICENSE_*`/`FORGE_*` env + 共享卷）。

**两种做法**：
- **A. 默认关闭可重启用**：加全局开关 `LICENSE_REQUIRED`（出厂 false），后端中间件直接放行、前端守卫/徽章/激活页全隐藏。保留代码，将来想商业化可一键开回。**改动小、可逆**。
- **B. 彻底删除**：按清单删 licensing 模块/中间件/验签库/激活页/守卫/徽章/gateway license/部署 env。**最干净但不可逆**。

> ⚠️ 根 `.agent` 有「License Gating 全局强制」硬规则，此处是你为**开源**显式豁免本项目。请在决策①确认是 A 还是 B。

## 八、额外建议（我调研 + 对产品的理解，建议一并纳入）

1. **引用可溯源**（NotebookLM 式）：答案的 [1][2] 可点跳原文对应页/块高亮（依赖解析保留 bbox）。
2. **多文档对比问答**：选多个源一起问。
3. **解析结果可纠错**：人工修正 OCR/表格错误后回写。
4. **解析进度可视化 + 失败重试**：源列表实时显示 rendering 进度、失败一键重试。
5. **KB 概览 tab**：统计 + 最近活动 + 快速入口。
6. **大文件流式上传 + 异步解析**：上传即返回、解析进度轮询（现在是同步解析，大文件上传会久等）。
7. **解析缓存/去重**：相同文件哈希命中复用，省 VL 成本。

## 需要你拍板（这几个决定整体怎么做）

1. **License**：A 默认关闭可重启用（推荐，可逆）/ B 彻底删除？
2. **解析引擎 VL 档位**：默认走哪档？高精（整页 VL，质量最好但每页一次 VL 调用、token 成本高）/ 标准（词法+VL图片，省）/ 让用户上传时选？
3. **预览页默认模式**：默认进「对话」还是「解析对比」？
4. **渲染策略**：先做渐进渲染（快、立即见效）还是直接上「按需渲染+虚拟列表」（工作量大但彻底秒开）？
5. **范围**：P0+P1 一起做，还是先 P0 上线再做 P1？
